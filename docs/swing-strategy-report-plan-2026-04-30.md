# 波段策略预期分析报告与落地计划

> 日期：2026-04-30  
> 范围：基于当前 `mo-stock-selection` v2.3 说明、评分文档、架构文档与核心代码  
> 定位：在现有 1-3 交易日短线选股系统之外，新增 5-20 交易日波段策略画像

---

## 1. 结论

当前项目更适合做“收盘后选出次日短线候选”，核心信号偏事件驱动和短期异动：

- `limit`：涨停、断板反包，偏 1-3 日情绪延续。
- `moneyflow`：当日与近 3 日主力流入，偏短期资金确认。
- `lhb`：龙虎榜席位，偏短线催化。
- `sector` / `theme`：行业与题材强度，既适合短线，也可作为波段主线来源。
- `AI`：用于解释和补充风险，但目前仍依赖规则层候选。

波段策略不能简单把持仓周期从 1-3 天拉长到 5-20 天。原因是短线系统会偏好“当天强、次日可能延续”的股票，而波段更关注：

- 趋势结构是否健康。
- 回撤后是否有承接。
- 板块主线是否持续。
- 量能是否温和放大而不是短线透支。
- 入场后是否有明确的止损、止盈和移动止盈规则。

建议新增一个独立策略画像：`swing`。它复用现有数据采集、存储、题材、资金、硬规则和报告能力，但新增趋势、回撤、波动率、持仓跟踪和回测模块。

## 2. 波段策略定义

### 2.1 持仓周期

建议定义为：

| 项目 | 建议值 |
|---|---:|
| 预期持仓 | 5-20 个交易日 |
| 首次评估窗口 | T+5 |
| 标准退出窗口 | T+10 / T+20 |
| 单票目标收益 | 8%-15% |
| 单票止损 | -5% 到 -8% |
| 盈亏比要求 | 至少 1.5:1 |
| 同时持仓数量 | 5-8 只 |
| 单票仓位 | 10%-20%，强信号不超过 20% |

以上是策略工程参数，不构成投资建议。实际阈值需要通过历史回测确认。

### 2.2 目标股票画像

波段候选不追求“明天必须涨”，而是寻找 1-4 周内有继续上行概率的股票：

1. 中短期均线多头或刚刚扭转。
2. 股价处在 20 日均线附近，或突破后首次缩量回踩。
3. 近 5-10 日主力资金净流入为正。
4. 所属行业或题材处于 3-10 日持续强势，而不是单日脉冲。
5. 日内没有极端透支，避免连续一字板、巨量冲高回落、当日涨停追高。
6. 非 ST、非次新、非停牌、无重大负面公告。

## 3. 当前项目可复用能力

### 3.1 可直接复用

| 模块 | 复用方式 |
|---|---|
| `DailyIngestor` | 继续采集 K 线、资金流、龙虎榜、行业、题材数据 |
| `daily_kline` | 计算 MA5 / MA10 / MA20 / MA60、20 日涨幅、ATR、回撤 |
| `daily_basic` | 用换手率、量比、市值过滤流动性 |
| `moneyflow` | 从 3 日扩展到 5 日 / 10 日滚动资金确认 |
| `sw_daily` | 行业强度从当日扩展到 5 日 / 10 日趋势 |
| `ths_daily` / `limit_concept_daily` / `ths_concept_moneyflow` | 题材持续性与资金确认 |
| `hard_reject` | ST、次新、涨停追高、跌停、停牌、负面公告仍适用 |
| `render_md` | 可扩展为波段报告模板 |
| `AI analyzer` | 可改 prompt，让 AI 输出波段逻辑、目标区间、持仓条件 |

### 3.2 不建议直接复用为主信号

| 现有维度 | 原因 | 波段处理 |
|---|---|---|
| `limit` | 涨停与断板反包偏短线情绪 | 降权，作为催化信号 |
| `lhb` | 龙虎榜偏 1-3 日博弈 | 降权，重点看机构席位净买与持续性 |
| 当日 `moneyflow` | 单日资金容易误导 | 扩展为 5 日 / 10 日累计与稳定性 |
| 当日 `sector` | 单日板块涨幅可能是脉冲 | 改为 5 日 / 10 日强度和回撤 |

## 4. 建议评分框架

### 4.1 新增波段权重

建议新增 `config/weights_swing.yaml`，不要污染现有短线权重。

```yaml
strategy:
  name: swing
  holding_days: [5, 20]

dimension_weights:
  trend: 0.25
  pullback: 0.15
  moneyflow_swing: 0.20
  sector_theme_swing: 0.20
  catalyst: 0.10
  risk_liquidity: 0.10
  sentiment: 0.00
```

说明：

- `trend` 是波段核心，权重最高。
- `pullback` 解决“怎么买”的问题，避免只买到高位加速。
- `moneyflow_swing` 从短线 3 日扩到 5/10 日。
- `sector_theme_swing` 把现有 `sector` 和 `theme` 做持续性合成。
- `catalyst` 复用 `limit` / `lhb`，但不再作为主权重。
- `risk_liquidity` 防止选到波动过大、流动性不足、短期透支的票。
- `sentiment` 未实现前建议为 0，避免固定分母压低可解释性。

### 4.2 维度设计

#### trend：趋势结构

目标：判断股票是否处于可持有的上升结构。

建议打分：

| 条件 | 加分 |
|---|---:|
| 收盘价 > MA20 | +20 |
| MA5 > MA10 > MA20 | +25 |
| MA20 斜率为正 | +20 |
| 收盘价 > MA60 或 MA20 上穿 MA60 | +15 |
| 近 20 日涨幅在 5%-35% | +20 |
| 近 20 日涨幅 > 60% | -30 |

关键点：波段不是买最强的当天，而是买趋势仍健康且没有严重透支的股票。

#### pullback：回踩承接

目标：找到趋势中的低风险介入点。

建议打分：

| 条件 | 加分 |
|---|---:|
| 近 5 日最大回撤 3%-12% | +25 |
| 收盘价距离 MA10 / MA20 不超过 5% | +25 |
| 回撤期间成交量低于上涨阶段均量 | +20 |
| 今日收盘重新站上 MA5 或 MA10 | +20 |
| 今日长上影且放巨量 | -30 |

当前 `daily_kline` 已有 OHLC 字段，上影线、实体强弱和回撤幅度都可以直接计算。

#### moneyflow_swing：波段资金

目标：确认资金不是一日游。

建议打分：

| 条件 | 加分 |
|---|---:|
| 近 5 日主力累计净流入 > 0 | +25 |
| 近 10 日主力累计净流入 > 0 | +25 |
| 近 5 日净流入天数 >= 3 天 | +20 |
| 大单 / 超大单净流入占比稳定为正 | +20 |
| 小单净买、大单净卖 | -30 |

现有 `MoneyflowFilter` 可复用公式和细节，但需要把窗口从 3 日扩展到 5/10 日。

#### sector_theme_swing：主线持续性

目标：确认不是单股孤立上涨，而是行业 / 题材有持续资金和相对强度。

建议打分：

| 条件 | 加分 |
|---|---:|
| 所属申万一级行业近 5 日涨幅 TOP 5 | +25 |
| 所属申万一级行业近 10 日涨幅 TOP 8 | +20 |
| 题材近 5 日排名靠前 | +25 |
| 题材资金净流入为正 | +15 |
| 行业和题材同时命中 | +15 |

现有 `SectorFilter` 和 `ThemeFilter` 是日频强度，波段版应改成多日强度，避免单日脉冲。

#### catalyst：短线催化

目标：保留短线系统的优势，但避免催化主导波段选股。

建议来源：

- 断板反包：有加分，但不能单独入选。
- 龙虎榜：机构席位净买优先于普通游资净买。
- 涨停：当日涨停仍不建议追高，只作为板块或强度信号。

建议权重不超过 10%。

#### risk_liquidity：风险与流动性

目标：把“能涨”和“能拿”分开。

建议打分：

| 条件 | 处理 |
|---|---|
| 20 日平均成交额过低 | 扣分或淘汰 |
| 近 20 日振幅过大 | 扣分 |
| 近 3 日涨幅过大且放量 | 扣分 |
| 当前价距离 MA20 超过 20% | 扣分 |
| ST / 次新 / 停牌 / 负面公告 | 继续硬淘汰 |

## 5. 策略预期

### 5.1 市场环境适配

| 市场环境 | 波段策略表现预期 |
|---|---|
| 指数震荡上行 | 最适合，趋势和回踩信号容易兑现 |
| 结构性题材行情 | 适合，重点看 `sector_theme_swing` |
| 单日脉冲行情 | 一般，波段版会主动降权追高信号 |
| 指数单边下跌 | 不适合，应降低仓位或空仓 |
| 极端缩量行情 | 一般，资金持续性会不足 |

### 5.2 期望指标

未回测前不要承诺收益，只设工程验证目标：

| 指标 | 初始目标 |
|---|---:|
| T+5 胜率 | >= 45% |
| T+10 胜率 | >= 48% |
| T+20 胜率 | >= 50% |
| 平均盈亏比 | >= 1.5 |
| 单票最大亏损 | 控制在 -8% 附近 |
| 策略最大回撤 | 需要回测确认，先以 12%-18% 为观察线 |
| Top 20 同板块上限 | 继续沿用 4 只 |

### 5.3 交易规则预案

入场：

- 收盘后出波段候选，次日不追高。
- 优先买入回踩 MA10 / MA20 后重新转强的股票。
- 如果次日高开超过 5%，等待盘中回落或放弃。

止损：

- 固定止损：买入价 -6% 到 -8%。
- 结构止损：有效跌破 MA20 且次日不能收回。
- 时间止损：T+5 仍未盈利且趋势转弱，退出或降仓。

止盈：

- 第一目标：+8% 到 +10%，可减仓。
- 第二目标：+15% 到 +20%，看板块强度决定是否保留。
- 移动止盈：盈利超过 10% 后，跌破 MA10 或从高点回撤 6%-8% 退出。

## 6. 工程落地计划

### Phase 0：策略配置与文档

目标：不改现有短线策略，先把波段作为独立 profile 接入。

任务：

1. 新增 `config/weights_swing.yaml`。
2. 新增策略参数：持仓周期、止损、止盈、回测窗口。
3. CLI 增加策略参数，例如：

```bash
mo-stock run-once --strategy short
mo-stock run-once --strategy swing --skip-ai
```

如果短期不想改 CLI，可以先新增：

```bash
mo-stock run-swing --date 2026-04-30
```

### Phase 1：波段规则层 MVP

新增文件建议：

| 文件 | 作用 |
|---|---|
| `src/mo_stock/filters/swing/trend_filter.py` | MA 结构、趋势斜率、20 日涨幅 |
| `src/mo_stock/filters/swing/pullback_filter.py` | 回踩 MA10 / MA20、缩量回撤、重新转强 |
| `src/mo_stock/filters/swing/moneyflow_swing_filter.py` | 5/10 日资金持续性 |
| `src/mo_stock/filters/sector_theme_swing_filter.py` | 行业 + 题材多日持续性 |
| `src/mo_stock/filters/swing/risk_liquidity_filter.py` | 流动性、波动率、透支度 |

测试建议：

- MA 多头排列得分。
- 周末 / 节假日窗口计算使用交易日，而不是自然日。
- 回撤到 MA20 后反弹得分。
- 近 5/10 日资金累计为正得分。
- 极端高位透支扣分。

### Phase 2：结果表与报告

当前 `selection_result` 没有策略字段。如果短线和波段同一天都要跑，建议二选一：

方案 A：给结果表增加 `strategy` 字段。

- 优点：复用现有表和报告逻辑。
- 缺点：需要 Alembic migration，唯一键也要调整为 `(trade_date, strategy, ts_code)`。

方案 B：新增独立表 `swing_selection_result`。

- 优点：不影响现有短线结果。
- 缺点：会有部分重复代码。

推荐方案 A，因为后续还可能有 `position`、`low_volatility`、`index_timing` 等策略画像。

报告建议输出：

```text
data/reports/swing/YYYY-MM-DD.md
data/reports/swing/YYYY-MM-DD.json
```

报告内容要和短线日报不同，新增：

- 建议观察周期：T+5 / T+10 / T+20。
- 趋势结构证据。
- 回踩位置。
- 止损价 / 结构止损条件。
- 第一目标位 / 第二目标位。
- 持仓跟踪状态。

### Phase 3：回测模块

新增波段回测是必须项，否则阈值只能靠经验。

建议新增：

| 文件 | 作用 |
|---|---|
| `src/mo_stock/backtest/swing.py` | 波段回测主逻辑 |
| `src/mo_stock/backtest/metrics.py` | 胜率、盈亏比、最大回撤、期望收益 |
| `tests/unit/backtest/test_swing_backtest.py` | 单元测试 |

回测指标：

- T+5 / T+10 / T+20 收益。
- 最大有利涨幅 `MFE`。
- 最大不利跌幅 `MAE`。
- 是否触发止损。
- 是否触发止盈。
- 持有期最大回撤。
- 行业集中度。

CLI 示例：

```bash
mo-stock backtest-swing --start 2025-01-01 --end 2026-04-30 --top-n 20
```

### Phase 4：AI 波段增强

当前 AI prompt 偏短线日报，可新增波段 prompt：

- 判断趋势是否可持续。
- 判断上涨来自主线还是单日异动。
- 输出 5/10/20 日观察条件。
- 输出结构止损，而不是只给次日止损价。
- 标记“只适合观察，不适合买入”的候选。

AI 不建议直接决定买卖，仍作为解释层和风险补充层。

## 7. 推荐优先级

| 优先级 | 事项 | 原因 |
|---|---|---|
| P0 | 新增 `weights_swing.yaml` 与策略 profile | 避免污染现有短线策略 |
| P0 | 新增 `TrendFilter` / `PullbackFilter` | 波段策略的核心，不可缺 |
| P0 | 回测 T+5/T+10/T+20 | 没有回测无法判断阈值是否有效 |
| P1 | 结果表增加 `strategy` 字段 | 支持短线和波段并行 |
| P1 | 波段报告模板 | 短线日报模板不适合持仓计划 |
| P1 | 资金流 5/10 日窗口 | 单日资金信号不足 |
| P2 | AI 波段 prompt | 提升解释质量，但不是第一优先级 |
| P2 | 持仓跟踪模块 | 回测稳定后再接入 |

## 8. MVP 验收标准

第一版波段策略可以按以下标准验收：

1. 不影响现有 `mo-stock run-once` 短线流程。
2. 能输出独立的波段候选报告。
3. 每只候选至少有 3 类有效证据：趋势、资金、行业 / 题材。
4. 报告中必须有入场条件、止损条件和观察周期。
5. 能跑最近 180 天回测，输出 T+5 / T+10 / T+20 指标。
6. 单元测试覆盖趋势、回踩、资金窗口、交易日窗口和硬规则。

## 9. 总体判断

这个项目已经有不错的波段策略基础：数据层、行业题材、资金流、硬规则、报告和 AI 框架都能复用。真正缺的是波段专属的趋势结构、回踩承接、持仓退出和回测评价。

建议不要把现有短线策略改造成波段策略，而是保留 `short`，新增 `swing`。这样两个策略可以共享数据，但评分逻辑、报告和回测目标彼此独立，后续也更容易比较哪套策略在不同市场环境下更有效。
