# 板块集中度治理：三重保障机制

> 版本：v2.4（2026-04-30），对应 commit `2439001` + `13b068e`

## 问题背景

A 股有板块联动效应——某天电子板块爆发时，同板块的几十只股票会在多个维度同时拿高分。
修复前的实盘数据：

| 日期 | 板块分布 | 状态 |
|------|----------|------|
| 04-24 | 801030 × 12 (60%) + 801050 × 6 (30%) | 严重集中 |
| 04-27 | 801080 × 20 (100%) | 单板块霸榜 |

Top 20 全部来自同一板块，本质上是"同一笔交易"，分散风险的目的完全失效。

### 根因分析

1. **sector 维度整板块统一加分**：TOP1 板块所有成分股 +70，是"白送分"，个股本身无需有任何亮点
2. **theme 维度与板块高度重叠**：热门概念（半导体/芯片）和热门板块（电子）几乎一一对应
3. **moneyflow 离散分档压缩区分度**：板块爆发日同板块大量股票同时跨过 1% 门槛，拿相同的 35 分
4. **旧版 limit_filter 有板块溢出**（v2.3 已修复）：给同板块所有非涨停股 +60，与 sector 多重共线性

## 三重保障机制

### 第一层：sector 降档（从源头压缩板块级加分）

**改动文件**：`src/mo_stock/filters/short/sector_filter.py`

**核心思路**：降低"仅凭板块归属就能拿高分"的幅度，让个股级维度（limit / moneyflow / lhb）的区分作用更大。

#### 降档前后对比

**排名加分 `_rank_to_bonus`**：

| rank | 旧值 | 新值 | 降幅 |
|------|------|------|------|
| 1 | 70 | 50 | -29% |
| 2 | 60 | 40 | -33% |
| 3 | 50 | 35 | -30% |
| 4 | 40 | 28 | -30% |
| 5 | 30 | 22 | -27% |

**趋势加分 `_three_day_avg_bonus`**：

| 3 日均涨幅 | 旧值 | 新值 |
|-----------|------|------|
| ≥ 5% | 30 | 20 |
| ≥ 2% | 15 | 10 |

维度满分从 100 降到 70（rank 50 + trend 20）。

#### 对 rule_score 的影响

```
rule_score = Σ(dim_score × weight) / Σ(全部权重)
```

sector 维度权重 0.10，降档前后对 rule_score 的最大贡献：

```
旧版：70 × 0.10 / 1.00 = 7.0 分（占理论上限 90 分的 7.8%）
新版：50 × 0.10 / 1.00 = 5.0 分（占理论上限 90 分的 5.6%，降幅 29%）
```

降档后，板块级"白送分"变少，个股信号（limit 0.25 + moneyflow 0.25 + lhb 0.20）的权重相对增大。

**类比**：把"考名校加 70 分"改成"考名校加 50 分"，让个人成绩更重要。

### 第二层：moneyflow 连续化（提升个股区分度）

**改动文件**：`src/mo_stock/filters/short/moneyflow_filter.py`

**核心思路**：同板块内的股票不再拿到相同的资金流分数，让排序能真正区分"板块内谁最强"。

#### 旧版：离散分档

```python
# 旧版 _today_bonus_tier（4 级离散）
if ratio_pct >= 5.0:  return 50    # 极强
if ratio_pct >= 1.0:  return 35    # 强
if ratio_pct >= 0.3:  return 20    # 中
return 5                          # 弱
```

板块爆发日的问题——同板块大量股票同时跨过门槛，拿到相同分数：

```
股票A 净流入 1.1%  → 35 分
股票B 净流入 4.9%  → 35 分   （几乎一样！）
```

#### 新版：线性插值

```python
# 新版 _today_bonus_tier（连续函数）
if ratio_pct < 0.3:  return 0
if ratio_pct >= 5.0: return 50.0
return 5.0 + (ratio_pct - 0.3) / 4.7 * 40.0
```

公式：`f(r) = 5 + (r - 0.3) / 4.7 × 40`，斜率 ≈ 8.5 分/1%。

同样的板块，不同个股的 moneyflow 分数拉开差距：

```
股票A 净流入 1.1%  → 5 + 0.8/4.7×40 ≈ 11.8 分
股票B 净流入 4.9%  → 5 + 4.6/4.7×40 ≈ 44.1 分
```

#### gate 逻辑优化

旧版用 `if today_bonus == 0: continue`，会把净流入 < 0.3% 但有 rolling/ratio 正信号的股票也排除。

新版改为：

```python
if net_mf_wan <= 0:
    continue  # 仅净流出跳过

# ... 计算 rolling_bonus / ratio_bonus ...

if clamp(score) <= 0:
    continue  # 最终 0 分不入 results
```

效果：净流入 < 0.3% 时 today_bonus=0，但仍可叠加 rolling/ratio 信号入榜。

**类比**：把"身高 170 以上都算高个子"改成"精确量身高按厘米排序"。

### 第三层：cap 硬上限（最终兜底）

**改动文件**：`src/mo_stock/scorer/combine.py`（v2.3 已实现，本次未修改）

**核心思路**：不管评分怎么算，最终 Top N 里单板块最多入选数有硬限制。

#### 配置

```yaml
# config/weights.yaml
combine:
  max_stocks_per_sector: 4        # 同一申万一级板块最多入选数
  max_unknown_sector_stocks: 0    # 无板块归属股不限（0=不限）
```

#### 运行逻辑

```python
# combine.py:336-360（简化）
for item in scored:                          # 按 final_score 降序遍历
    sector = sector_map.get(item["ts_code"])
    sector_count = sector_counts.get(sector, 0)

    over_sector_cap = max_per_sector > 0 and sector_count >= max_per_sector

    picked = (
        reject_reason is None           # 未被硬规则淘汰
        and not over_sector_cap          # 未达板块上限
        and picked_rank < effective_top_n  # 未超 TOP N
    )

    if picked:
        sector_counts[sector] += 1       # 仅入选股消耗名额
```

04-30 实际效果：电子板块 4 只入选（cap 触顶），宝明科技（rule=15.5）、华灿光电（rule=17.1）
等电子股被拦下，名额让给其他板块。

#### UNKNOWN 处理

`max_unknown_sector_stocks: 0` 表示不限。设计理由：

- UNKNOWN 股之间没有板块关联，不存在集中风险
- 限制它们等于因数据缺失而误淘汰优质股
- 真正的防线是 `combine.py:324` 的 WARNING 日志（提醒 `refresh-basics`）

**类比**：高考录取时"每省最多录取 4 人"，不管某省考生考得多好。

## 三层协同流程

```
原始股票池（4876 只）
    │
    ▼
第一层：sector 降档
    │  板块白送分从 7.0 降到 5.0（-29%）
    │  个股级信号权重相对增大
    │
    ▼
第二层：moneyflow 连续化
    │  同板块内个股分数产生梯度（11.8 vs 44.1）
    │  不再"过了门槛都是同一分"
    │
    ▼
按 final_score 排序
    │
    ▼
第三层：cap 硬上限
    │  单板块最多 4 只（max_stocks_per_sector=4）
    │  超出的标记 REJECTED，名额让给其他板块
    │
    ▼
最终 Top 20：9 个板块，无板块超 20%
```

**为什么三层缺一不可**：

| 只用一层 | 问题 |
|---------|------|
| 只用 cap | 丢弃高分优质股（如电子板块 rule=17.1 的股被拦） |
| 只用降档 | 极端行情下仍可能集中（所有维度同时给同板块加分） |
| 只用连续化 | 其他维度（limit/theme/lhb）仍是离散的，板块联动仍在 |

三层从不同角度发力：**第一层降信号强度**，**第二层提区分度**，**第三层硬截断**，
组合使用才能在各种市场环境下稳定分散。
